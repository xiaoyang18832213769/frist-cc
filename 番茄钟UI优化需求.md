# 番茄钟 UI 优化需求文档

**版本：** v1.0  
**目标读者：** Claude Code / 开发 Agent  
**项目路径：** `D:\桌面\AI羊\Frist-cc`  
**主文件：** `pomodoro.pyw`（桌面版，优先优化）、`pomodoro.html`（可选同步）  
**优化目标：** 在保留现有功能的前提下，将 UI 提升至「大厂级产品」观感，采用莫兰迪色系，并支持多主题切换。

---

## 1. 项目背景与现状

### 1.1 现有功能（必须保留，不得破坏）

| 功能 | 说明 |
|------|------|
| 三阶段计时 | 专注工作 / 短休息 / 长休息 |
| 环形进度 | Canvas 圆弧进度（桌面版）/ SVG 圆环（网页版） |
| 番茄计数 | 当前周期内已完成番茄数（🍅 图标） |
| 今日统计 | 当日完成番茄总数 |
| 配置持久化 | `pomodoro_config.json`（桌面）/ localStorage（网页） |
| 快捷键 | 空格开始/暂停、R 重置、S 跳过、Esc 关闭设置 |
| 设置项 | 工作/短休/长休时长、长休间隔、提示音、窗口置顶 |
| 阶段结束 | 提示音、窗口闪烁提醒（桌面版） |
| 启动方式 | 双击 `pomodoro.pyw` 或 `启动番茄钟.bat` |

### 1.2 现有 UI 问题

1. **配色偏艳**：当前红 `#e74c3c`、绿 `#27ae60`、蓝 `#2980b9`，饱和度高，缺乏高级感
2. **Tkinter 原生感强**：按钮、Spinbox、边框样式偏系统默认，层次不足
3. **无主题系统**：颜色硬编码在 `COLORS` 字典，无法切换
4. **信息层级弱**：模式标签、时间、操作区视觉权重区分不明显
5. **动效不足**：阶段切换、按钮反馈、倒计时最后几秒提示较生硬
6. **窗口尺寸固定**：`330x430`，设置面板展开时可能拥挤

## 2. 设计目标

### 2.1 总体方向

> **「安静、克制、有呼吸感」的专注工具**  
> 参考 Apple 时钟 / Notion 极简 / Linear 精致 / Microsoft Fluent 柔和圆角，但不做花哨装饰。

### 2.2 核心原则

| 原则 | 说明 |
|------|------|
| **莫兰迪色系** | 低饱和、灰调、柔和，长时间看不疲劳 |
| **多主题可选** | 至少 4 套主题，一键切换并持久化 |
| **阶段色语义** | 工作/短休/长休各有独立语义色，但同属同一色系家族 |
| **大厂级细节** | 统一间距、圆角、阴影、字重、动效曲线 |
| **专注优先** | 主界面只突出「时间 + 一个主操作」，次要功能收起 |
| **无障碍** | 对比度 WCAG AA，不仅靠颜色区分状态 |

## 3. 莫兰迪主题系统（核心要求）

### 3.1 主题架构

实现 `THEMES` 配置对象，每套主题包含完整 Design Token：

```python
THEMES = {
  "theme_id": {
    "name": "主题显示名",
    "work":       { "main", "light", "subtle", "text", "ring_bg" },
    "shortBreak": { "main", "light", "subtle", "text", "ring_bg" },
    "longBreak":  { "main", "light", "subtle", "text", "ring_bg" },
    "bg": "...",           # 窗口/页面背景
    "card": "...",         # 卡片背景
    "text": "...",         # 主文字
    "text_secondary": "...",
    "muted": "...",        # 辅助文字
    "border": "...",       # 分割线/边框
    "surface": "...",      # 次级面板背景
    "btn_secondary": "...",# 次要按钮背景
    "shadow": "...",       # 阴影色（或 rgba）
    "accent": "...",       # 强调色（设置、选中态）
  }
}
```

### 3.3 主题切换 UI 要求

- 入口：设置面板内增加 **「外观主题」** 区域
- 形式：**色块预览卡片**（4 个小圆角方块横排），选中态有描边 + 对勾
- 切换效果：**300ms 全局颜色过渡**（`transition`），禁止闪烁
- 持久化：写入 `pomodoro_config.json` 的 `"theme": "misty_rose"` 字段
- 即时生效：切换后所有控件（按钮、圆环、标签、面板）同步更新

### 3.4 配色约束（莫兰迪规范）

1. 主色饱和度建议 **15%–35%**，禁止使用纯红 `#FF0000`、纯绿等
2. 背景与卡片色差 **≤ 8%**，保持柔和层次
3. 文字与背景对比度 **≥ 4.5:1**（WCAG AA）
4. 同一主题内，work / short / long 三色 **色相不同但明度、饱和度接近**
5. 禁止使用高饱和渐变；背景可用极淡的单色渐变（5% 以内色差）

## 4. 布局与信息架构

### 4.1 主界面结构（自上而下）

```
┌─────────────────────────────┐
│  [模式标签]  专注工作         │  ← 胶囊标签，当前阶段色
│  第 2 / 4 个番茄             │  ← 次要说明，muted 色
│                              │
│         ╭─────────╮          │
│        │  24:58   │         │  ← 视觉焦点，最大字号
│         ╰─────────╯          │     环形进度环绕
│       🍅 🍅 ○ ○               │  ← 番茄进度，已完成/未完成区分
│                              │
│    [↺]  [ ▶ 开始 ]  [⏭]     │  ← 主按钮居中突出
│                              │
│         ⚙ 设置               │  ← 弱视觉权重
│  今日完成 3 个番茄            │  ← 底部统计
└─────────────────────────────┘
```

### 4.2 窗口尺寸

| 状态 | 尺寸建议 |
|------|----------|
| 主界面收起设置 | `360 × 480` px（比现版略大，留呼吸感） |
| 设置面板展开 | 高度自适应展开至 `~580` px，或设置区可滚动 |
| 最小宽度 | 不低于 `320` px |

### 4.3 间距系统（8pt 网格）

| Token | 值 |
|-------|-----|
| space-xs | 4px |
| space-sm | 8px |
| space-md | 16px |
| space-lg | 24px |
| space-xl | 32px |

所有 padding、margin 必须是 4 的倍数。

## 5. 组件级 UI 规范

### 5.1 字体

| 用途 | 字体 | 字号 | 字重 |
|------|------|------|------|
| 倒计时数字 | Consolas / JetBrains Mono | 48–52px | Bold |
| 模式标签 | Microsoft YaHei UI | 12px | Semibold |
| 辅助说明 | Microsoft YaHei UI | 11px | Regular |
| 主按钮 | Microsoft YaHei UI | 14px | Semibold |
| 设置项 | Microsoft YaHei UI | 11px | Regular |
| 统计 | Microsoft YaHei UI | 10px | Regular |

- 数字必须使用 **等宽数字**（tabular nums）
- 禁止倒计时切换时数字跳动错位

### 5.2 圆角

| 组件 | 圆角 |
|------|------|
| 主卡片/窗口内容区 | 20px |
| 主按钮 | 14px |
| 次要按钮（圆形） | 50% |
| 模式标签胶囊 | 20px |
| 设置面板 | 16px |
| 输入框 Spinbox | 10px |
| 主题预览卡片 | 12px |

### 5.3 进度环

- 线宽：**6–8px**，圆角端点（round cap）
- 底色环：主题 `ring_bg`，约 `#E8E6E4` 级别
- 进度环：当前阶段 `main` 色
- 动画：每秒更新时 **ease-out 200ms** 过渡（若 Tkinter 难以实现 CSS 过渡，至少保证无锯齿）
- 可选增强：进度环末端加 4px 圆点指示器（参考 Apple Fitness）

### 5.4 主按钮（开始/暂停）

参考 **Apple / Linear** 风格：

- 高度：**44px**（符合触控最小点击区域）
- 最小宽度：**120px**
- 背景：当前阶段 `main` 色
- 文字：白色或对比色（自动计算，确保可读）
- Hover/Active：明度 ±8%
- 暂停态：按钮文案「⏸ 暂停」，颜色不变
- 禁用态：opacity 0.4

### 5.5 次要按钮（重置 ↺、跳过 ⏭）

- 尺寸：**40 × 40** px 圆形
- 背景：`btn_secondary`
- 图标：muted 色，hover 时变 `text` 色
- Tooltip：悬停 500ms 后显示「重置 (R)」「跳过 (S)」

### 5.6 模式标签

- 形式：胶囊 Badge，非纯文字
- 内容：`● 专注工作`（用圆点替代 emoji，更高级；或保留 🍅 但降低饱和度）
- 背景：当前阶段 `light`
- 文字：当前阶段 `text`
- 阶段切换：**200ms 背景色渐变**

### 5.7 番茄进度指示

**方案 A（推荐）：** 莫兰迪色圆点

- 未完成：空心圆或浅灰实心 `#D8D4D0`
- 已完成：当前主题 work.main 色实心圆 + 轻微放大动画

**方案 B：** 保留 🍅 emoji，但未完成态 `grayscale + opacity 0.3`

### 5.8 设置面板

参考 **Notion / Apple 设置** 风格：

- 背景：`surface` 色，1px `border` 描边
- 每行：左标签 + 右控件，行高 **36px**
- 数字输入：自定义 Spinbox 样式，隐藏默认上下箭头或改为细线箭头
- 开关：iOS 风格 Toggle（宽 44px，高 26px），开启色为当前阶段 main 或 accent
- 分区标题：「计时」「提醒」「外观」三组，组间距 16px
- 新增分组 **「外观」**：主题选择器

### 5.9 最后 5 秒提示

- 当前有 `flash_bg` 闪烁，改为更克制的方式：
  - 卡片边框或进度环轻微 pulse（opacity 0.6↔1.0，1s 周期）
  - **禁止** 整屏剧烈闪烁

## 8. 技术实现要求

### 8.1 桌面版 `pomodoro.pyw`

1. 将 `COLORS` 重构为 `THEMES` + `get_theme()` + `apply_theme()`
2. 所有 `configure(bg=..., fg=...)` 统一走主题函数，禁止散落硬编码色值
3. Tkinter 限制下尽量模拟现代 UI：
   - 使用 `tk.Frame` 模拟卡片阴影（浅色边框 + 略深外框）
   - 按钮 `relief="flat"`, `bd=0`, `cursor="hand2"`
   - 可考虑 `tkinter.ttk` 的 `clam` 主题并自定义 Style
4. 配置新增字段：

```json
{
  "theme": "misty_rose",
  "work": 25,
  "short": 5,
  "long": 15,
  "interval": 4,
  "sound": true,
  "topmost": true
}
```

5. **不得引入** 需 pip 安装的重型 UI 库（保持双击即用）
6. 若 Tkinter 无法实现某动效，优先保证配色与布局，动效可降级

### 8.2 网页版 `pomodoro.html`（可选同步）

- CSS 变量与桌面版主题 Token 一一对应
- `body[data-theme="misty_rose"]` 切换
- 保持单文件、无外部依赖

### 8.3 兼容性

- Windows 10+
- Python 3.x 自带 tkinter
- 窗口置顶、提示音、配置读写逻辑不变
